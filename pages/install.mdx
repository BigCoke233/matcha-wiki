# 安装和使用

这篇教程将会讲述基础的安装 Typecho 程序和使用 Matcha 主题的方法，如果你已经掌握了这些，请跳过这一部分。

## 安装 Typecho

Matcha 主题是为 Typecho 设计的博客主题，只能在 Typecho 上使用，所以你必须先安装 Typecho。

### 环境要求

Typecho 对环境的要求如下：

- PHP 5.1 以上
- Mysql, PostgreSQL, SQLite 任意一种数据库支持，并在 PHP 中安装了相关扩展
- CURL 或者 Socket 扩展支持
- mbstring 或者 iconv 扩展支持

但鉴于你使用的是 Matcha 主题，我们建议你按照以下的要求配置环境：

- PHP 7.0.0 以上
- CURL 拓展支持

其他的与 Typecho 要求相同。

### 获取程序本体

在 [Typecho 官网](http://typecho.org/) 下载最新 1.2.0 稳定版，尽管 Matcha 主题在旧的版本也能够正常运行，但我们建议你使用最新的稳定版，因为 Matcha 主题在开发过程中不会考虑旧版本的兼容性。

### 开始安装

将你获取的文件解压，将如下结构的文件放入你的网站根目录。

```text
/admin/
/install/
/usr/
/var/
/index.php
/install.php
```

访问你的网站，根据提示进行安装。

## 使用 Matcha 主题

Matcha 主题的安装步骤与其他大多数主题无异。

### 获取主题文件

在主题仓库的 [Releases](https://github.com/BigCoke233/matcha/releases) 页面下载最新版主题的 zip 压缩包，或者直接下载仓库的 `main` 分支。

需要特别注意的是，在 Releases 页面（即 发行包页面）下载到的主题版本为**发行版**或**先行版**：版本号形如 `1.0.0` 的版本为发行版；版本号形如 `1.1.0-beta.1` 的为先行版。
在仓库页面（即主题的 github 主页）直接下载到的版本称作**发行版**，发行版也有自己的版本号，即 `当前最新发行版版本号-alpha+最近一次 commit 的哈希码前七位`，不过大部分时候不会使用。

**稳定性上，发行版 > 先行版 > 开发版；内容新旧程度上，开发版 > 先行版 > 发行版。**我们建议尽量使用发行版或先行版，开发版可能会有未被发现的 bug。

### 安装并启用主题

将获到的主题文件解压，确保文件夹名称为 `matcha`，如果不是，请重命名。

将 `matcha` 文件夹放入 Typecho 安装目录下的 `usr/themes/` 目录，然后登陆后台，点击 控制台-外观，找到 Matcha 主题，点击启用即可。
